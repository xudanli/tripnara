数据表说明
行程模板类
journey_templates：行程模板主表，保存模板名称、封面、核心亮点、默认安全提示、背景等 JSON 配置，可复用生成多个用户行程。
template_days：模板内每日结构，记录 day_number、标题、摘要以及当天详细的 JSON 描述，支撑模板拆分至每日粒度。
template_time_slots：模板的具体时段活动，包括类型、标题、开始时间、活动详情、推荐交通方式等字段，用于在模板层定义日程细节。
preparation_profiles：为不同目的地或主题提供的准备任务模板，包含携带物品、安全提醒、注意事项等内容，可与模板关联。

行程实例类
journeys：实际用户行程主表，关联 user_id 与 journey_template_id，包含旅程概要、核心洞察、生成配置、行程任务 JSON 等。
journey_days：用户行程的每日信息，记录具体日期、主题标题、当天概览以及详细的 JSON 描述，与 journeys 形成一对多关系。
journey_time_slots：行程中的时段活动（如上午、下午、晚上），保存活动类型、摘要、交通信息、媒体资源、来源方式等。
journey_edits：行程编辑历史日志，记录修改类型、原值/新值、操作人、时间戳，便于审计与版本回溯。
journey_feedback：用户对行程或生成结果的评分与文字反馈，可用于迭代模板或调整生成策略。
journey_shares：行程分享记录，包含分享链接、访问次数、分享渠道等，用于追踪分享传播效果。

用户与账号
users：系统基础账号表，保存登录凭据（邮箱/手机号）、昵称、头像、语言偏好、状态等。
user_profiles：用户扩展画像信息，包括国籍、常住地、常旅偏好、个性标签等，与 users 一对一。
user_auth_providers：第三方登录绑定记录，如 Apple、Google、微信，包含 external_id、provider、token 信息。
user_settings：用户个性化配置，涵盖通知开关、功能实验、隐私偏好等。

AI 生成与请求追踪
ai_request_logs：每次调用 LLM/生成服务的日志，记录模块、Prompt、Response 摘要、耗时、token 统计，用于追踪成本和排查问题。
ai_generation_jobs：异步生成任务流水线，包含任务状态、步骤、执行耗时、错误信息以及关联的旅程 ID。
ai_safety_notice_cache：安全提示缓存表，按目的地 + 语言 + 文本摘要生成 Key，缓存最新安全提示文本与有效期。

外部数据与配置
destinations：目的地/城市/景点基础信息，含地理坐标、所属国家、别名、分类标签等。
countries：国家信息表，记录签证要求、货币、风险等级、入境政策等数据。
transport_modes：标准化交通方式字典（如步行、地铁、出租车等），用于前后端统一枚举。
media_assets：媒体资源元数据，保存图片/视频的 CDN 路径、版权方、授权类型等。
external_api_keys：外部服务访问密钥配置，通常只在后端可见，包含服务名称、键值、使用配额等。

系统通知与消息
notifications：站内通知记录，关联用户并存储通知类型、标题、内容、发送状态等。
